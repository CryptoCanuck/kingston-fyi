## Services

### PostgreSQL (supabase-db)
- **Image**: `supabase/postgres:15.1.1.61`
- **Purpose**: Primary database with PostGIS for geospatial queries
- **Port**: 5432 (internal only)
- **Extensions**: uuid-ossp, pgcrypto, pgjwt, postgis, postgis_topology

### GoTrue (supabase-auth)
- **Image**: `supabase/gotrue:v2.143.0`
- **Purpose**: Authentication and authorization
- **Port**: 9999 (internal)
- **Features**: Email/password, OAuth, JWT tokens, MFA

### PostgREST (supabase-rest)
- **Image**: `postgrest/postgrest:v12.0.1`
- **Purpose**: RESTful API auto-generated from database schema
- **Port**: 3000 (internal)
- **Features**: Automatic CRUD endpoints, filtering, pagination

### Storage API (supabase-storage)
- **Image**: `supabase/storage-api:v0.46.4`
- **Purpose**: File uploads and management
- **Port**: 5000 (internal)
- **Features**: S3-compatible, RLS integration, image transformations

### Kong (supabase-kong)
- **Image**: `kong:2.8.1`
- **Purpose**: API Gateway and routing
- **Port**: 8000 (internal API), 8443 (internal HTTPS)
- **Features**: Rate limiting, authentication, routing

### pg-meta (supabase-meta)
- **Image**: `supabase/postgres-meta:v0.75.0`
- **Purpose**: Database metadata and Studio backend
- **Port**: 8080 (internal)
- **Features**: Schema inspection, database management

### Studio (supabase-studio) - Optional
- **Image**: `supabase/studio:20240101-8e4a094`
- **Purpose**: Web-based database management UI
- **Port**: 3000 (can be exposed for dev)
- **Features**: Table editor, SQL editor, API docs

## Configuration Files

### kong.yml
API Gateway routing configuration. Routes requests to appropriate backend services:
- `/auth/v1/*` вҶ’ GoTrue
- `/rest/v1/*` вҶ’ PostgREST
- `/storage/v1/*` вҶ’ Storage API
- `/pg/*` вҶ’ pg-meta

### init.sql
PostgreSQL initialization script:
- Enables PostGIS extension
- Creates required schemas (auth, storage, extensions)
- Sets up initial permissions

## Environment Variables

Required variables (auto-generated by scripts/generate-secrets.sh):

```bash
# Database
POSTGRES_PASSWORD=<generated>
POSTGRES_HOST=supabase-db
POSTGRES_PORT=5432
POSTGRES_DB=postgres

# JWT Configuration
JWT_SECRET=<32-byte-hex>
JWT_EXPIRY=3600

# API Keys
ANON_KEY=<jwt-token>
SERVICE_ROLE_KEY=<jwt-token>

# Service URLs
GOTRUE_DB_DRIVER=postgres
GOTRUE_API_HOST=0.0.0.0
GOTRUE_API_PORT=9999
GOTRUE_SITE_URL=http://localhost:3000
GOTRUE_URI_ALLOW_LIST=*

# Storage
STORAGE_BACKEND=file
FILE_SIZE_LIMIT=52428800
```

## JWT Key Generation

Supabase uses JWT tokens for authentication. Generate them with:

```bash
# JWT_SECRET (32 bytes hex)
openssl rand -hex 32

# ANON_KEY (JWT token with anon role)
# Use https://supabase.com/docs/guides/self-hosting/docker#generate-api-keys
# Or use the generate-secrets.sh script

# SERVICE_ROLE_KEY (JWT token with service_role)
# Same as above, but with service_role claims
```

## Connecting from Next.js

```typescript
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!, // http://kong:8000 (internal)
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)
```

## Data Persistence

Volumes are configured for:
- `supabase-db-data`: PostgreSQL data
- `supabase-storage-data`: Uploaded files

## Security Considerations

1. **Never expose PostgreSQL** directly to the internet
2. **Keep JWT_SECRET secure** - compromise means full access
3. **Use RLS (Row Level Security)** for all tables
4. **SERVICE_ROLE_KEY bypasses RLS** - only use server-side
5. **ANON_KEY is public** - safe to use client-side with RLS

## Upgrading

1. Check [Supabase releases](https://github.com/supabase/supabase/releases)
2. Update image tags in docker-compose.yml
3. Backup database before upgrading
4. Test in development first

## Troubleshooting

### Database won't start
- Check logs: `docker logs supabase-db`
- Verify volume permissions
- Ensure PostGIS image is correct

### Auth not working
- Verify JWT_SECRET matches in all services
- Check GoTrue logs: `docker logs supabase-auth`
- Ensure GOTRUE_SITE_URL matches your domain

### Storage uploads failing
- Check storage volume permissions
- Verify FILE_SIZE_LIMIT
- Check storage API logs: `docker logs supabase-storage`
